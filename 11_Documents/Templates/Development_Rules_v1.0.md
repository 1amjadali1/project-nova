# Development Rules

## General

- Use TypeScript everywhere.
- Never push directly to main.
- Every feature uses its own branch.
- Every Pull Request requires review.
- Never hardcode secrets.
- Write readable code before clever code.
- Prefer composition over duplication.
- Every API must be documented.
- Every database change must use migrations.
- Every bug fix should include a test whenever practical.

## Code Style

- Use ESLint
- Use Prettier
- Use meaningful variable names
- Keep functions focused on a single responsibility
- Avoid deep nesting

## Documentation

If code changes architecture or behavior, update the related documentation before merging.
