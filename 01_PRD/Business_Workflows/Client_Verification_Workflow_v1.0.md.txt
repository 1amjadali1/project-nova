Client Verification Workflow v1.0
Purpose

This document defines the complete lifecycle of a background verification case inside Project Nova, from the moment a client creates a verification request until the final report is delivered.

Actors
Client

Creates verification requests.

Uploads candidate documents.

Tracks verification progress.

Downloads reports.

Operations Executive

Reviews documents.

Performs manual verification.

Initiates external verification requests.

Updates verification status.

Team Leader

Assigns verification cases.

Reviews completed checks.

Performs quality assurance.

Approves or returns cases.

Operations Manager

Monitors SLA.

Views analytics.

Handles escalations.

Approves exceptional cases.

System

Creates case IDs.

Stores documents securely.

Maintains audit logs.

Triggers notifications.

Calculates SLA timers.

Generates reports.

Records every activity.
